# Design Concept

## 컨셉 이름

AI Service Flow Map

## 설명

포트폴리오 전체가 하나의 서비스 설계 지도처럼 보여야 한다.

방문자가 스크롤하면 조아름이라는 개발자가
사용자의 문제를 발견하고,
AI 기능을 설계하고,
백엔드와 프론트엔드를 연결하고,
실제 사용자가 쓸 수 있는 서비스 경험으로 완성하는 과정을 따라가게 된다.

즉, 포트폴리오는 단순한 페이지가 아니라
“AI 서비스 흐름 지도”처럼 느껴져야 한다.

## 비주얼 메타포

### 1. Service Map
프로젝트, 기술, 역할, 문제 해결 과정이 지도 위의 노드처럼 연결된다.

### 2. AI Pipeline
User Input → AI Reasoning → API Bridge → UI Preview → Confirm & Save  
이 흐름이 전체 디자인에서 반복적으로 나타난다.

### 3. Warm Technical Lab
차갑고 사이버틱한 AI 느낌이 아니라,
따뜻한 종이, 설계 노트, 지도, 회로, 서비스 플로우가 섞인 느낌.

### 4. Personal Signature
대표 프로젝트인 “내일도와줘! 몽글마을”의 따뜻한 감성은 유지한다.
다만 전체를 귀엽게 만들지 말고,
작은 스티커, 스탬프, 썸네일 정도의 시그니처 요소로만 사용한다.

## 디자인 규칙

- 흰 카드만 반복하지 않는다.
- 지도 위의 노드, 라인, 라벨, 패널이 겹쳐진 느낌을 만든다.
- 각 섹션은 독립된 박스가 아니라 하나의 flow line으로 연결되어야 한다.
- 카드 형태는 다양하게 사용한다.
  - passport card
  - node card
  - note card
  - island panel
  - toolkit box
  - response panel
- 단, 컬러와 타이포그래피는 통일한다.
